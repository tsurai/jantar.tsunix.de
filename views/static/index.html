Truly open source

Powered by Github Jantar is completly free and open source. Feel free to clone, fork, contribute, watch and star.
The MIT Licence fully embraces the open source spirit and gives developer the freedom they need and deserve.

Light but sane

  • RESTful pattern with protection against cross-site request forgery
  • Secure default settings for TLS
  • Safe default HTTP header
  • Encrypted and signed cookies using AES256 and HMAC(SHA256)
  • Compatible with http.HandlerFunc

Built with love

Jantar is a passion project by a single person. Without deadlines Jantar can grow gracefully and evolve with changing needs and requirements.
While trying to stay light, the main focus is on security, stability and performance.

More security by default

Nowadays security is the most crucial aspect of the World Wide Web.
Frameworks should take responsibility and offer protection against the most common vulnerabilities right out of the box

SSL/TLS

No SSL

SSL 2.0 is broken in many ways and should never be used. Even SSL 3.0, while being better than 2.0 cannot be validated under FIPS 140-2 because of its weak key derivation process.
That's why Jantar only accepts TLS 1.0 or higher.

Strong prefered cipher

Jantar provides strong prefered cipher by default to provide the best possible protection possible.
Some example cipher are:

  • ECDHE ECDSA with AES-256 GCM SHA-384
  • ECDHE RSA with AES-256 GCM SHA-384
  • RSA with AES-256 CBC SHA-256
  • RSA with AES-128 CBC SHA-256

Responsive to critical bugs

Some bugs like Heartbleed are so critical that they require immediate actions taken. Jantar is always up to date and will warn you about any unusual events that need your attention.
